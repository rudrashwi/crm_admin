WRITE_CALL_LOG PERMISSION FIX
Version 1.0.1+2 - February 12, 2026
========================================

🚨 ISSUE FOUND:
Google Play Console showed both READ_CALL_LOG and WRITE_CALL_LOG permissions, even though WRITE_CALL_LOG was not explicitly declared in AndroidManifest.xml

🔍 ROOT CAUSE:
The `permission_handler_android` package (used for runtime permission requests) automatically includes ALL possible Android permissions in its manifest, including ones you don't use. During the Gradle build process, these get merged into your app's final manifest.

Permissions automatically added by permission_handler:
- ❌ android.permission.WRITE_CALL_LOG (not needed - we only READ)
- ❌ android.permission.READ_PHONE_NUMBERS (not needed)
- ❌ android.permission.READ_MEDIA_IMAGES (not needed)
- ❌ android.permission.READ_MEDIA_VIDEO (not needed)
- ❌ android.permission.READ_MEDIA_AUDIO (not needed)

✅ SOLUTION:
Added explicit permission removal directives in AndroidManifest.xml using `tools:node="remove"` to prevent these unwanted permissions from being merged into the final app.

========================================
## Changes Made to AndroidManifest.xml:
========================================

ADDED these lines after READ_CALL_LOG declaration:

```xml
<!-- Explicitly remove WRITE_CALL_LOG added by permission_handler -->
<uses-permission android:name="android.permission.WRITE_CALL_LOG" tools:node="remove"/>

<!-- Remove other unwanted permissions from permission_handler -->
<uses-permission android:name="android.permission.READ_PHONE_NUMBERS" tools:node="remove"/>
<uses-permission android:name="android.permission.READ_MEDIA_IMAGES" tools:node="remove"/>
<uses-permission android:name="android.permission.READ_MEDIA_VIDEO" tools:node="remove"/>
<uses-permission android:name="android.permission.READ_MEDIA_AUDIO" tools:node="remove"/>
```

## Why This Happened:
========================================

The `permission_handler` package is designed to support ALL Android permissions so developers can request any permission they need at runtime. However, this means it declares ALL possible permissions in its Android manifest.

During the Gradle build process:
1. Your app's AndroidManifest.xml is read
2. All dependency manifests (including permission_handler) are merged
3. By default, ALL permissions from ALL dependencies are included
4. Result: Your app ends up with permissions you never declared

The solution is to explicitly REMOVE unwanted permissions using:
```xml
<uses-permission android:name="unwanted.permission" tools:node="remove"/>
```

This tells Gradle: "Even if a dependency declares this permission, DO NOT include it in my final app."
